# 🤖 Multi-Modal RAG Assistant (Travel & Lifestyle Context)

### 🌍 Обзор проекта
Высокотехнологичный ИИ-ассистент на базе архитектуры **RAG (Retrieval-Augmented Generation)**, предназначенный для персонализированного планирования долгосрочного пребывания на Шри-Ланке. Система объединяет обработку текстовых данных, компьютерное зрение (Vision) и распознавание речи (Speech-to-Text).

Бот спроектирован для работы в условиях жестких ограничений (бюджет, специфические требования к инфраструктуре) и обеспечивает высокую точность ответов за счет интеграции локальной базы знаний и инструментов веб-поиска.

### 🏗 Архитектурная концепция
Данный проект представляет собой универсальный **RAG-фреймворк**. Несмотря на то, что текущая реализация настроена на Шри-Ланку, архитектура системы является доменно-независимой.

* **Масштабируемость:** замена папки контекстных данных (`/data`) и системного промпта позволяет адаптировать ассистента под любую индустрию (LegalTech, HR-tech, техподдержка) без изменения программного кода ядра.
* **Гибридный поиск:** интеллектуальное переключение между локальным контекстом и глобальной сетью (Tavily API).

### 🛠 Ключевые архитектурные решения

* **🚀 Advanced RAG Pipeline:** реализован многоэтапный поиск информации:
    * **L1 Cache:** система кэширования на базе JSON с хэшированием контекста (**MD5**). Снижает латентность и затраты на API на 40-60%.
    * **L2 Vector Search:** поиск по локальной векторной базе (**ChromaDB**) с оптимизированным параметром `k=5` для обеспечения высокой релевантности.
    * **L3 Web-Search Integration:** динамический вызов **Tavily API** в случае отсутствия данных в локальном контексте.

* **📸 Multi-Modal Integration:** единая точка входа для всех типов данных. Изображения анализируются через Vision-модели и сопоставляются с требованиями пользователя (например, верификация рабочего места на фото жилья), а голосовые сообщения транскрибируются для бесшовного диалога.

* **🧠 Context & Memory Management:** система управления состоянием (**State Management**) сохраняет глубину памяти до **20 последних сообщений**, обеспечивая консистентность ответов.

* **📊 Quality Assurance (Ragas):** проведена автоматизированная оценка качества. Текущие показатели:
    * **Faithfulness (Верность источнику):** `0.80` — подтверждает отсутствие галлюцинаций.
    * **Answer Relevancy:** `0.95` — гарантирует высокую точность попадания в запрос.

### 💻 Стек технологий
* **Core Logic:** Python, LangChain, OpenAI (GPT-4o-mini, Vision, Whisper)
* **Database:** ChromaDB (Vector Store)
* **Interface:** Aiogram 3.x (Telegram Bot)
* **QA & Analytics:** Ragas, Pandas, Datasets (используются для валидации системы)

### 🛡 Оптимизация и безопасность
* **Performance Tuning:** реализована стратегия очистки кэша **LRU (Least Recently Used)**, предотвращающая деградацию $O(n)$ при масштабировании базы запросов.
* **Production-Ready Config:** управление конфигурацией и секретами реализовано через **Environment Variables** (библиотека `python-dotenv`).
